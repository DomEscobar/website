## Raw ideas

### 1. Daily AI podcast that runs while you sleep

- Source: Bronson Elliott — “I Built a Daily AI Podcast That Runs While I Sleep” — https://bronsonelliott.substack.com/p/i-built-a-daily-ai-podcast-that-runs
- Why unusual: It turns newsletters/emails into an actual podcast feed: Gmail search → synthesis → local Qwen3-TTS → MP3 + transcript → GitHub Pages feed. The surprising part is that a non-developer built a production automation through conversation.
- Who it helps: Normal people with too much reading; developers who want a reusable synthesis/audio pipeline.
- Workflow: label input sources → scheduled 5 AM run → synthesize 3–4 real stories → generate audio → publish private/public feed → listen during commute/walk.
- Prompt/tool: “Read everything in this label since the last run. Synthesize the 3–4 real stories, ignore repeated noise, and create an 8-minute conversational briefing.”
- Risk / when not to use: Audio can hide nuance; not suitable for legal/financial decisions without source links and transcript.
- Score: 7/7
- Could become post: yes

### 2. Orchestrator-only life assistant with background sub-agents

- Source: Hector Flores — “I Repurposed a Coding Agent as a Life Assistant. Then My Twins Came 10 Weeks Early.” — https://htek.dev/articles/coding-agent-as-life-assistant-nicu
- Why unusual: The main chat agent does almost nothing except route. Incoming requests from Telegram or cron are delegated to background sub-agents with their own context windows, so one stuck task does not block the assistant.
- Who it helps: Developers building serious personal agents; normal people indirectly through more reliable assistants.
- Workflow: incoming request → orchestrator classifies → launches domain sub-agent → sub-agent uses its own tools/memory → report back.
- Prompt/tool: “Do not solve this in the main session. Pick the correct domain agent, delegate the work, and return only the final result or blocker.”
- Risk / when not to use: Invisible background autonomy can become risky without logs, permissions, and clear stop rules.
- Score: 7/7
- Could become post: yes

### 3. Local AI for private automations you would not trust ChatGPT with

- Source: How-To Geek — “3 things I automate with local AI that I'd never trust ChatGPT with” — https://www.howtogeek.com/things-i-automate-with-local-ai-that-id-never-trust-chatgpt-with/
- Why unusual: The value is privacy-first mundane automation: receipts into budgeting CSVs, voice transcription, and local tool access via MCP, all on a local model.
- Who it helps: Normal people with sensitive personal data; developers who want local model workflows.
- Workflow: local LM Studio model → MCP tools for files/apps → private input like receipts/notes → structured local output.
- Prompt/tool: “Extract merchant, date, total, category, and confidence from this receipt. Append only high-confidence rows to my local CSV; send uncertain rows to review.”
- Risk / when not to use: Local models can be less reliable; private does not mean correct.
- Score: 6/7
- Could become post: yes

### 4. Family tech-support bot with a narrow delegation boundary

- Source: Mejba Ahmed — “I Built My Family Tech-Support Bot With OpenClaw in 2026” — https://www.mejba.me/blog/openclaw-personal-ai-assistant-2026-build
- Why unusual: Instead of “AI assistant for everything,” it picks one annoying recurring task: family tech support. It answers the simple 80% and routes the rest with context.
- Who it helps: Normal people who become unofficial tech support; developers who want a tiny assistant project with clear ROI.
- Workflow: family sends Telegram voice/text → bot classifies common problem → replies in the owner’s voice/style → escalates hard issues with context.
- Prompt/tool: “If this is printer/Wi-Fi/password reset basics, answer. If it requires account access or payment, summarize and route to me.”
- Risk / when not to use: Voice cloning can confuse boundaries; must be clear it is an assistant, not the human.
- Score: 6/7
- Could become post: yes

### 5. GitHub Agentic Workflows as recurring repo hygiene

- Source: GitHub Blog — “Automate repository tasks with GitHub Agentic Workflows” — https://github.blog/ai-and-ml/automate-repository-tasks-with-github-agentic-workflows/ and gh-aw release notes https://github.com/github/gh-aw/releases/tag/v0.71.6
- Why unusual: Agents move from interactive coding sessions into GitHub Actions with sandboxing, safe outputs, approval gates, and recurring workflows for issue triage, docs, tests, and repo reports.
- Who it helps: Developers/teams with repo maintenance debt.
- Workflow: Markdown workflow spec → compile to GitHub Actions → agent runs in sandbox → safe output creates comment/PR/report → human reviews risky changes.
- Prompt/tool: “Every Monday, inspect flaky CI failures, summarize likely root causes, and open a draft issue with evidence links. Do not edit code.”
- Risk / when not to use: Cost and permissions matter; start with read-only reports/comments before PR creation.
- Score: 6/7
- Could become post: yes

## Best candidate angle

Title: **Turn Your Newsletter Pile Into a Daily AI Podcast**

Short answer: Don’t ask AI to summarize one email at a time. Label the sources, run a scheduled synthesis, and make an audio briefing you can actually consume away from the screen.

Flow:

```txt
newsletter label
→ scheduled run
→ synthesis, not summaries
→ local TTS
→ MP3 + transcript
→ podcast feed
```

Visual component idea:
- `FlowGraphic` for the pipeline
- `CompareGrid` for “email summaries” vs “audio synthesis feed”
- `DoDont` for source-heavy decisions vs casual briefing
- `KeyPoints` for synthesis rules: dedupe, conflicts, source links, transcript

## Candidate prompt

```txt
Read all emails in [label] since the last run.
Do not summarize each one.
Synthesize the briefing:
1. 3-4 real stories or patterns
2. what is new vs repeated
3. conflicting claims or uncertainty
4. source links for each story
5. conversational script for an 8-minute audio briefing
```
