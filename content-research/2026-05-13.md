## Raw ideas

### 1. The AI that runs a household (open-sourced)

- Source: Hector Flores — https://htek.dev/articles/copilot-home-assistant-ai-runs-my-household — repo at https://github.com/htekdev/copilot-home-assistant
- Why unusual: Not a demo. A real multi-agent system (17 agents) running on GitHub Copilot CLI + Telegram that manages groceries, pregnancy reminders, dog feeding, home maintenance, budget tracking, and a 4-year-old's curriculum — for a real family. Entirely open-source.
- Who it helps: Developers who want to build a personal AI assistant; normal people intrigued by what a real household AI looks like in 2026.
- Workflow: Define agent as Markdown file → agent reads constitution + memory → runs on cron or Telegram trigger → executes via Copilot CLI → saves learnings → every correction is permanent.
- Prompt/tool: "I have a recurring household task: [describe]. Design a simple agent persona that handles this — domain rules, memory format, integration points."
- Risk / when not to use: Requires GitHub Copilot subscription; Copilot CLI can execute shell commands, so untrusted agent definitions are a security risk.
- Score: 7/7
- Could become post: yes

### 2. agentcmd — AI workflow orchestration platform

- Source: https://agentcmd.dev/
- Why unusual: Visual workflow builder for multi-agent AI coding pipelines. Define slash commands, chain Claude + Codex + Gemini in one workflow, watch real-time execution. "Stop babysitting your AI agents."
- Who it helps: Developers using coding agents who want to orchestrate complex multi-step workflows.
- Workflow: Define spec → agentcmd generates → implement → review → PR — all unattended with real-time WebSocket streaming.
- Prompt/tool: "Build a workflow: generate spec → implement with Codex → review with Claude → create PR. Run in parallel where possible."
- Risk / when not to use: Lock-in risk; still early-stage tool.
- Score: 5/7
- Could become post: yes

### 3. aitasks — agentic IDE in your terminal

- Source: https://www.aitasks.io/
- Why unusual: A full IDE experience in tmux with Kanban board, code browser, agent monitoring, and AI-enhanced git — all terminal-native. Task decomposition across parallel git worktrees with multiple agents.
- Who it helps: Terminal-first developers working with code agents on complex tasks.
- Workflow: Open ait ide → define task → auto-explode into child tasks → run agents in parallel isolated worktrees → review via per-language guides.
- Prompt/tool: No prompt needed — it's a tmux-native tool.
- Risk / when not to use: Terminal-only; steep learning curve if you're not already in tmux.
- Score: 5/7
- Could become post: yes

### 4. Vuel.ai — personal AI that does the work (not just chat)

- Source: https://www.vuel.ai/
- Why unusual: Makes phone calls (navigates IVRs, wait queues, talks to humans), files receipts via photo OCR, drafts and sends emails. "Not a chatbot — an assistant that finishes things."
- Who it helps: Normal people drowning in admin tasks (pharmacy refills, reservations, expense tracking).
- Workflow: Snap receipt → Vuel reads/filed. Ask Vuel to call pharmacy → navigates IVR → talks to human → reports back.
- Prompt/tool: Not applicable (product, not prompt technique).
- Risk / when not to use: Early access only; privacy concerns with phone calls; calling real businesses on your behalf is high-stakes.
- Score: 5/7
- Could become post: yes

### 5. vchaindz/workflow — n8n for the command line

- Source: https://github.com/vchaindz/workflow (Rust, single binary, 42 templates)
- Why unusual: Like n8n but in your terminal. AI-native: Claude Code / Codex CLI / Gemini CLI can generate, fix, and refine workflows from natural language. Parallel DAGs, sub-workflows, webhook REST API, encrypted secrets. No Docker, no web server.
- Who it helps: Developers who want workflow automation without the Docker stack.
- Workflow: Write YAML workflow → run with single binary → AI auto-fixes on failure → TUI or headless CLI.
- Prompt/tool: Not needed — tool itself is AI-native.
- Risk / when not to use: Early-stage (v0.4.2, single contributor); YAML can get complex for branching logic.
- Score: 4/7
- Could become post: yes

## Best candidate angle

Title: **The AI That Actually Runs a Household (And It's Open Source)**

Why: This is the most unusual + practical combination. A real family uses 17 AI agents (Markdown files!) on GitHub Copilot CLI to run their entire household — meal planning, budget, pregnancy reminders, dog care, home maintenance, and an ADHD-aware task coach. The full system is open source. It's not a demo.

Flow:

```txt
annoyance → define agent persona → markdown file → run on cron/Telegram → agent acts → learns from corrections → improves over time
```

Visual component idea:
- `FlowGraphic` for the agent lifecycle (define → load → act → learn → persist)
- `KeyPoints` for what each of the 17 agents does
- `CompareGrid`: "AI chatbot" vs "household agent system"
- `DoDont`: "Define clear domain boundaries" vs "Let agents cross into each other's territory"

## Candidate prompt

```txt
I want to build an AI agent for [household task].
Design a simple agent persona:

1. Domain expertise (what it knows)
2. Decision rules (when to act, when to ask)
3. Memory format (what to save between runs)
4. Integration points (Telegram, calendar, files, etc.)
5. One standing order it should never forget
```
