## Die kurze Version

Der schlechteste Prompt für UI-Agenten ist: **“Mach das schöner.”**

Nicht weil KI kein Frontend kann. Sondern weil “schön” kein ausführbarer Test ist. Ein Agent kann Tailwind schreiben, Komponenten umbauen und SVGs säubern. Aber wenn die QA nur aus Bauchgefühl besteht, bekommst du Bauchgefühl als Output.

Die bessere Frage ist:

> Welche Teile dieses Designs kann ich messen?

## Der Trick: “sieht falsch aus” übersetzen

Ein Mensch sagt: “Der Abstand wirkt komisch.”

Eine gute Pipeline sagt:

- erwarteter Abstand: `32px`
- tatsächlicher Abstand: `24px`
- erwartete Schriftgröße: `18px`
- tatsächliche Schriftgröße: `16px`
- Safari unterstützt diese Animation nicht sauber

Das ist der Unterschied zwischen Meinung und Feedback, mit dem ein Agent arbeiten kann.

## Mini-Workflow zum Klauen

```txt
Implementiere diese UI-Sektion.
Verlasse dich nicht nur auf optische Ähnlichkeit.

Gib zurück:
1. extrahierte Designwerte: Spacing, Type, Größe, Farben, Radius
2. Komponentenplan
3. Browser-computed-style-Checks nach Umsetzung
4. konkrete Abweichungen zum Design
5. was noch menschlichen Taste-Review braucht
```

## Wann lohnt sich das?

Diese Pipeline ist nicht für jeden Button nötig. Sie lohnt sich, wenn du ähnliche UI-Arbeit mehrfach machst: Landingpage-Sektionen, Designsystem-Migrationen, Figma-to-Code, Kundenwebsites oder Komponenten, die wirklich konsistent aussehen müssen.

## Mensch bleibt wichtig

Messbare QA ersetzt keinen guten Designer. Sie nimmt nur das Nervige aus dem Review: “Ist der Abstand korrekt? Ist die Schriftgröße richtig? Bricht Safari?”

Danach kann der Mensch wieder das tun, was Menschen besser können: entscheiden, ob das UI wirklich gut ist.
